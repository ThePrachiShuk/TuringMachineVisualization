01 — Automata Theory

What is Automata Theory?

Automata Theory is a branch of computer science and mathematics that studies abstract machines — mathematical models of computation — and the problems these machines can solve. It provides the theoretical backbone for understanding what computers can compute, how efficiently they can compute it, and — crucially — what lies beyond the reach of any computer.

⬡

Finite Automata

The simplest machine model. Accepts or rejects strings based on a fixed set of states. Recognises regular languages.

⬡

Pushdown Automata

Extends finite automata with a stack, allowing recognition of context-free languages such as balanced parentheses.

⬡

Turing Machines

The most powerful model. Equivalent in computational power to any modern computer, capturing the full extent of what is computable.

⬡

Computability Theory

Studies which problems are solvable by algorithms at all — the Halting Problem being the canonical example of an unsolvable problem.

02 — The Turing Machine

What is a Turing Machine?

Introduced by Alan Turing in 1936, the Turing Machine is a theoretical model of computation that defines the fundamental limits of what can be computed. Despite its simplicity, it is capable of performing any calculation that any modern computer can perform. It consists of four core elements working in concert.

A Turing Machine is formally defined as the 7-tuple (Q, Σ, Γ, δ, q₀, q_accept, q_reject) — a finite set of states, input alphabet, tape alphabet, transition function, start state, accept state, and reject state.

03 — Components

Machine Components

01

Tape

The tape serves as the machine's unbounded memory. It is divided into discrete cells, each holding exactly one symbol from a defined tape alphabet Γ. The tape extends infinitely in both directions and is initially populated with the input string, with all remaining cells containing the blank symbol (□).

02

Read / Write Head

The head is always positioned over exactly one tape cell. At each step of computation it reads the symbol in the current cell, writes a new symbol (which may be the same), then moves either one cell to the left (L) or one cell to the right (R). Head movement and writes happen simultaneously in a single atomic step.

03

States

The machine possesses a finite set Q of internal states. Each state represents a discrete stage of the computation. At any given moment the machine is in exactly one state. The current state, combined with the symbol under the head, completely determines what the machine does next.

04

Start State

One designated state q₀ ∈ Q is the start state. When the machine is initialised with an input string, the head is placed on the leftmost symbol of that string and the machine enters q₀ to begin computation.

05

Accept / Halting States

Certain states are designated as halt states (q_accept or q_reject). When the machine transitions into a halt state, computation stops immediately. The machine then either accepts or rejects the input depending on which halting state was reached. Not all machines halt.

06

Transition Function

The transition function δ : Q × Γ → Q × Γ × {L, R} is the machine's "program". Given the current state and the symbol under the head, it specifies the next state, the symbol to write, and the direction of head movement. If no rule applies for the current (state, symbol) pair, the machine halts immediately.

05 — Busy Beaver Numbers

The Busy Beaver Problem

Among all halting Turing Machines with exactly n states and a binary tape alphabet, the Busy Beaver machine is the one that writes the most 1s on the tape before halting. The Busy Beaver function Σ(n) gives the maximum number of 1s any such machine can write.

Σ(n) grows faster than any computable function. Even defining Σ(5) precisely required decades of distributed computation, and Σ(6) remains unknown. This makes the Busy Beaver function fundamentally uncomputable.

States (n) Σ(n) — Max 1s written S(n) — Max steps before halt Champion machine
1 1 1 Trivially computable
2 4 6 BB(2) — known since 1962
3 6 21 BB(3) — known since 1965
4 13 107 BB(4) — known since 1983
5 4098 47,176,870 BB(5) — confirmed 2024
6 ≥ 10↑↑15 unknown BB(6) — currently unknown

Why does this matter? The Busy Beaver function is intimately connected to the Halting Problem. If you could compute Σ(n) for all n, you could decide whether any Turing Machine halts — but Turing proved the Halting Problem is undecidable. Therefore Σ(n) must itself be uncomputable. This illustrates a deep truth: even the simplest computational systems — machines with just a handful of states — can produce behaviour that is fundamentally beyond the reach of any algorithm to predict.
