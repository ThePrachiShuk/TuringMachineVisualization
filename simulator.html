Busy Beaver Exploration

Explore the Busy Beaver function — machines that maximize ones written before halting. This connects directly to the Halting Problem and the limits of computability.

Why Busy Beaver is Uncomputable

No algorithm can compute Σ(n) for all n. If it could, we could solve the Halting Problem: run any n-state machine for Σ(n) steps — if it hasn't halted, it never will.

BB(5) takes over 47 million steps while only using 6 tape cells — demonstrating how deceptively simple rules can lead to astronomically complex behavior.
